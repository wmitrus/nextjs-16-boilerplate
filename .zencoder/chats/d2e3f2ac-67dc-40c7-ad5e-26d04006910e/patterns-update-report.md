## Changes Made

### 1. New Entry — SEC-11: SDK Client Cache Key Completeness

**File updated**: `docs/ai/general/SECURITY_CODING_PATTERNS.md`

Added a full SEC-11 entry covering:

- Pattern to avoid: module-level SDK client cache keyed by subset of configuration (e.g., only `clientKey`, ignoring `apiHost`)
- Why it's dangerous: silent wrong-backend selection; no error thrown; incorrect feature flag evaluation
- Correct pattern: composite cache key including all differentiating config — `${clientKey}|${apiHost}`
- Relationship to SEC-09: SEC-09 covers mutable SDK attribute state; SEC-11 covers incomplete cache key selection
- Separator guidance: use `|` for SDK client key + URL host composites

**Index table updated** in `docs/ai/general/SECURITY_CODING_PATTERNS.md` — SEC-11 row added.

### 2. Updated SEC-08 — SQLint False Positive Note

**File updated**: `docs/ai/general/SECURITY_CODING_PATTERNS.md`

Added a scanner false positive note under SEC-08: SQLint's ANSI SQL rule incorrectly flags `UNIQUE NULLS NOT DISTINCT(...)` syntax in Drizzle-generated migration files. This is valid PostgreSQL 15+ syntax generated by `.nullsNotDistinct()`. Do not edit generated migrations. Suppress SQLint warning or configure PostgreSQL dialect.
